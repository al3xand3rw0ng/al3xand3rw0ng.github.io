Marble Solitaire

In collaboration with a partner, I implemented three versions of marble solitaire: English, European, and Triangular. Marble solitaire follows a simple ruleset. A player can move a marble, horizontally or vertically, by making a jump over one adjacent marble into an empty slot. When all slots are vacant except for the center, the player has won. The player loses if no other moves are possible and the condition identified above is not met.

This assignment required using object-oriented design principles to limit code duplication. Additionally, this project was created by following the Model-View-Controller (MVC) design pattern.

The primary issue faced in our implementation was adapting our design to work with other versions of marble solitaire. To do this, we created abstract classes to create inheritable and abstract methods. This additionally improved the readability of our code.
